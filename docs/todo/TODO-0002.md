# TODO-0002 — micro_entity: record codec (lossless markdown <-> entity)
ADR: ADR-0003, ADR-0004
Status: active — 4 items (0 done, 4 todo)

Settled decisions for this batch (from session clarification):
- Storage format is schema-strict Markdown + YAML frontmatter, one file per entity.
- Hand-editable / LOSSLESS: a machine write MUST preserve key order, comments, and
  body formatting (ruamel.yaml round-trip). Round-trip tests required.
- A record file = YAML frontmatter block + a body region after the closing `---`.
  Frontmatter holds the attribute bag + the structural timestamps; the body region
  is the Entity `body` field (ADR-0004), preserved verbatim.
- `id` is NEVER stored in frontmatter — it is the filename stem, supplied by the
  store layer (TODO-0003). Codec functions that build an Entity take the id as an
  argument.
- `created` / `updated` are the only structural keys lifted out of frontmatter; every
  other key is a generic attribute. (Timestamp MIGRATION from a `date` key is an adr
  PROFILE concern in TODO-0006, not here — this codec expects created/updated present.)

This batch is the pure text<->parts<->Entity codec; ALL functions are pure (no I/O).
File I/O, partitioning, and the clock live in the store (TODO-0003).

## 1. Parse a record document into frontmatter + body — [todo]
Goal: split raw markdown-with-frontmatter text into a round-trip frontmatter mapping
and the body region, losslessly.
Interface: in `src/micro_entity/codec.py`:
- `def parse_document(text: str) -> tuple[CommentedMap, str | None]` — uses
  ruamel.yaml round-trip load on the frontmatter block; returns the mapping plus the
  body text after the closing `---` (None if no body region exists). Raises a
  `CodecError(ValueError)` on malformed input (no opening `---`, unterminated block,
  frontmatter that is not a mapping).
Acceptance: frontmatter + body → (mapping, body); frontmatter only → (mapping, None);
an empty body region after `---` → (mapping, ""); comments and key order are retained
in the returned mapping (assert by re-emitting); missing/unterminated/non-mapping
frontmatter raises CodecError.
Constraints: ruamel.yaml round-trip API only; no Entity import here (pure text↔parts).
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
File scope: create `src/micro_entity/codec.py` and `tests/test_codec.py` only.

## 2. Serialize frontmatter + body back into a document — [todo]
Goal: render (frontmatter mapping, body) back to document text such that parse and
serialize are inverses on well-formed input (byte-stable round-trip), preserving key
order and comments.
Interface: in `src/micro_entity/codec.py`:
- `def serialize_document(frontmatter: CommentedMap, body: str | None) -> str`.
Acceptance: `serialize_document(*parse_document(text)) == text` for a hand-formatted
fixture with comments and a multi-line body (lossless); changing one value in the
mapping re-emits with all other keys, comments, and the body untouched; body None →
frontmatter-only output (no trailing body block); body "" → an empty body region.
Constraints: ruamel round-trip emit; pure.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
File scope: edit `src/micro_entity/codec.py` and `tests/test_codec.py` only.

## 3. Build an Entity from id + frontmatter + body — [todo]
Goal: map parsed parts into a validated Entity — lift the structural timestamps out
of frontmatter, treat every remaining key as an attribute, attach the body.
Interface: in `src/micro_entity/codec.py`:
- `def entity_from_parts(id: str, frontmatter: Mapping[str, object], body: str | None) -> Entity`
  — reads `created` and `updated` (ISO-8601 strings or datetimes) into tz-aware
  datetimes; all other frontmatter keys become `attributes`; constructs and returns
  an `Entity`. Propagates the Entity's validation errors (FormError / ValidationError)
  unchanged so the caller (store) can quarantine. Raises `CodecError` if `created` or
  `updated` is missing or unparseable.
Acceptance: a well-formed frontmatter + id → Entity with correct timestamps, the
non-timestamp keys as attributes, and the body set; `created`/`updated` accept both
an ISO string and a datetime; missing/unparseable timestamp → CodecError; an
invalid attribute value (nested) propagates the Entity validation error; the id is
validated by the Entity (one bad-id case).
Constraints: reuse `Entity` (TODO-0001) and `parse_document` types; do not duplicate
validation. `created`/`updated` are the ONLY structural keys; `id` is never read from
frontmatter.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
File scope: edit `src/micro_entity/codec.py` and `tests/test_codec.py` only.

## 4. Render a (new) Entity to frontmatter + body — [todo]
Goal: the CREATE direction — build a fresh frontmatter mapping + body from an Entity.
(Used only for brand-new records; lossless UPDATE patches the existing mapping in the
store and does NOT rebuild from the Entity.)
Interface: in `src/micro_entity/codec.py`:
- `def entity_to_parts(entity: Entity) -> tuple[CommentedMap, str | None]` — emits
  `created` and `updated` as ISO-8601 strings, then every attribute key/value;
  `id` is NOT emitted (it is the filename); returns (frontmatter, entity.body).
Acceptance: round-trips an Entity through `entity_to_parts` →
`serialize_document` → `parse_document` → `entity_from_parts` back to an equal Entity;
`id` does not appear in the emitted frontmatter; timestamps emit as ISO strings; body
None and body "" both round-trip.
Constraints: pure; produce a ruamel `CommentedMap` so the store can later patch it
losslessly. Attribute insertion order deterministic (insertion order of the dict).
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
File scope: edit `src/micro_entity/codec.py` and `tests/test_codec.py` only.
