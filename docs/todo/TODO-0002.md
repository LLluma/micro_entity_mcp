# TODO-0002 — micro_entity: frontmatter round-trip (lossless)
ADR: ADR-0003
Status: active — 2 items (0 done, 2 todo)

Settled decisions for this batch (from session clarification):
- Storage format is schema-strict Markdown + YAML frontmatter, one file per entity.
- Hand-editable / lossless: a machine write MUST preserve key order, comments, and
  body formatting (use ruamel.yaml round-trip). Round-trip tests required.
- A record file = YAML frontmatter block + a body region after the closing `---`.
  Frontmatter holds structural fields + the attribute bag; the body region is the
  Entity `body` field (ADR-0004), preserved verbatim.
- `id` is NOT stored in frontmatter — it is the filename stem (set by the store
  layer in TODO-0003, not here).

## 1. Parse a record document into its parts — [todo]
Goal: split raw markdown-with-frontmatter text into (frontmatter mapping, body
string), losslessly, using ruamel.yaml round-trip loading.
Interface: `def parse_document(text: str) -> tuple[CommentedMap, str | None]` in
`src/micro_entity/markdown.py` — returns the ruamel round-trip frontmatter mapping
and the body region (None if no body present). Raises a parse error on malformed
frontmatter (no opening `---`, unterminated block, non-mapping frontmatter).
Acceptance: frontmatter + body parses to mapping + body text; frontmatter-only (no
body) → body None; comments and key order are retained in the returned mapping;
malformed/absent frontmatter raises; an empty body region after `---` → "".
Constraints: ruamel.yaml round-trip API only; no Entity coupling here (pure
text↔parts). Full spec/interface refinement when this item is reached.
File scope: create `src/micro_entity/markdown.py`, `tests/test_markdown.py` only.

## 2. Serialize parts back into a record document — [todo]
Goal: render (frontmatter mapping, body) back to text such that
parse∘serialize and serialize∘parse are identity on well-formed input (lossless
round-trip), preserving key order and comments.
Interface: `def serialize_document(frontmatter: CommentedMap, body: str | None) -> str`.
Acceptance: round-trips a hand-formatted file byte-stable (key order + comments +
body intact); a status value changed in the mapping re-serializes with everything
else untouched; body None → frontmatter-only output. Full spec when reached.
File scope: edit `src/micro_entity/markdown.py`, `tests/test_markdown.py` only.
