## 2. Attribute value form-validation — [done]
Goal: a pure validator that enforces the generic attribute value shape the store
accepts — a scalar or a flat list of scalars, nothing nested. This is the "validate
form only, assign no meaning" rule for attribute VALUES.
Interface: in `src/micro_entity/validation.py`:
- `Scalar = str | int | float | bool` (module-level type alias).
- `class FormError(ValueError)` — raised on any malformed value.
- `def validate_attribute_value(value: object) -> None` — returns None if `value`
  is a `Scalar` or a `list` whose every element is a `Scalar`; otherwise raises
  `FormError` with a message naming the offending shape.
Acceptance:
- scalars pass: `"x"`, `""`, `0`, `3.14`, `True`;
- a flat list of scalars passes: `["a", "b"]`, `[1, 2]`, `[]` (empty list ok);
- nested/structured values raise FormError: a `dict`, a `list` containing a `list`,
  a `list` containing a `dict`, a `None`, a `tuple`, a set.
Note: `bool` is a `Scalar` deliberately (YAML true/false); do NOT special-case it
out even though `bool` is a subclass of `int`.
Constraints: pure function, no I/O, stdlib + the type alias only. Do not import
Pydantic here.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: the store validates the FORM of attributes only (well-typed:
scalar or list of scalars) and assigns no meaning to keys or values. (ADR-0001,
ADR-0002)
File scope: create `src/micro_entity/validation.py` and
`tests/test_validation.py` only. Do not touch anything else.

## 3. Id form-validation — [done]
Goal: a pure validator that enforces the filesystem-safe, backend-portable id form.
Kept a separate function (not inlined in the Entity model) so it mirrors the other
validators and can be reused by the store/profile layers.
Interface: in `src/micro_entity/validation.py` (extend the existing module):
- `def validate_id(value: str) -> None` — returns None if `value` matches
  `^[A-Za-z0-9_-][A-Za-z0-9._-]*$` (first char not a dot; remaining chars ASCII
  alphanumerics, dot, underscore, hyphen) AND `len(value) <= 200`; otherwise raises
  `FormError` naming the bad id.
Acceptance:
- valid ids pass: `ADR-0007`, `001`, `build-entity`, `3f9a_b2`;
- rejected (FormError): empty string; whitespace-containing; a path separator (`/`
  or `\`); `.` and `..`; a leading-dot name (`.hidden`); non-ASCII / emoji; a
  control character; an id of length 201.
Constraints: pure function, no I/O. Reuse `FormError` from item 2. Do not import
Pydantic.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: the server validates that an id is well-formed; ids are
filesystem-safe and portable across any storage backend. (ADR-0001, ADR-0004)
File scope: edit `src/micro_entity/validation.py` and `tests/test_validation.py`
only. Do not touch anything else.

## 4. Registered value-set validation — [done]
Goal: a pure, generic validator that checks an attribute's value(s) against a
caller-provided allowed set. This is the single mechanism behind both status enums
and any other cross-cutting attribute that must avoid typo-drift; the core knows no
specific attribute names or values. Status is merely its first consumer — this
function has no notion of "status".
Interface: in `src/micro_entity/validation.py` (extend the existing module):
- `def validate_against_set(value: Scalar | list[Scalar], allowed: collections.abc.Set[Scalar]) -> None`
  — returns None if every element of `value` is in `allowed` (a scalar is treated as
  a one-element set; a list must have all elements in `allowed`); otherwise raises
  `FormError` naming the offending value(s) and listing the allowed values.
Acceptance:
- a scalar present in the allowed set passes; a scalar absent raises FormError;
- a list whose every element is in the set passes; a list with any element absent
  raises FormError naming that element;
- an empty list value passes (no element violates the set — vacuously true);
- an empty allowed set: a scalar or non-empty list always raises FormError; an empty
  list value still passes (nothing to check);
- membership is type-strict and exact (consistent with item 2's Scalar typing): e.g.
  `1` (int) is not matched by an allowed set containing only `"1"` (str);
- the error message includes the offending value(s) and the allowed values.
Constraints: pure function, no I/O. Do not bake in any concrete attribute names or
values (no "status", "done", "Accepted", etc.) — both the value and the allowed set
are always passed in by the caller. Reuse the `Scalar` alias from item 2.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: a per-attribute registered value-set is generic form
validation applied to whatever attribute the caller chooses; the core assigns no
meaning to the attribute or its values. Status enums are one instance of this
mechanism, configured and enforced by the profile servers, never by the core.
(ADR-0001, ADR-0002, ADR-0004)
File scope: edit `src/micro_entity/validation.py` and `tests/test_validation.py`
only. Do not touch anything else.

## 5. Entity model — [done]
Goal: the generic entity as a validated Pydantic model that WIRES the existing
validators: an id, two timestamps, an optional prose body, and a bag of generic
attributes. There is NO structural `status` field — status, when a profile uses it,
is an ordinary key in the attribute bag (ADR-0004). The model validates FORM only
and reuses the item 2 / item 3 validators rather than reimplementing them.
Interface: in `src/micro_entity/entity.py`, a Pydantic v2 model `Entity` with:
- `id: str` — validated via `validate_id` (item 3) in a field validator; do NOT
  re-encode the regex here.
- `created: datetime` and `updated: datetime` — timezone-aware datetimes; a naive
  (tz-less) datetime is rejected.
- `body: str | None = None` — optional free-text prose (the document body region).
  NOT an attribute and NOT run through attribute validation: any string (incl. "")
  or None is accepted. Kept distinct so membership-query sees only attributes and
  full-text search sees only the body.
- `attributes: dict[str, Scalar | list[Scalar]]` — every value validated via
  `validate_attribute_value` (item 2) in a field validator; attribute keys must be
  non-empty strings.
- Immutable: `model_config = ConfigDict(frozen=True)`.
Acceptance:
- a well-formed Entity constructs and exposes its fields;
- a malformed id is rejected (delegated to `validate_id` — confirm one rejecting
  case and one passing case; the full id matrix is item 3's job, not retested here);
- empty attribute key rejected;
- `body` defaults to None when omitted; a string body (incl. "") is accepted verbatim
  and never appears in `attributes`;
- a nested dict/list attribute value is rejected (delegated to `validate_attribute_value`);
- a naive datetime for created/updated is rejected;
- the instance is frozen: attribute assignment after construction raises.
Constraints: reuse `validate_id`, `validate_attribute_value`, and `Scalar` from
`micro_entity.validation`; do not duplicate any validation logic. No I/O.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: an entity = an id + store-managed timestamps + an optional
prose body + a bag of generic attributes (key -> scalar | list); the server
validates form only and assigns no meaning to attribute keys or values; no
structural status field. (ADR-0001, ADR-0002, ADR-0004)
File scope: create `src/micro_entity/entity.py` and `tests/test_entity.py` only.
Do not modify `validation.py`; do not touch anything else.
