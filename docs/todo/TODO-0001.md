## 2. Attribute value form-validation тАФ [todo]
Goal: a pure validator that enforces the generic attribute value shape the store
accepts тАФ a scalar or a flat list of scalars, nothing nested. This is the "validate
form only, assign no meaning" rule for attribute VALUES.
Interface: in `src/micro_entity/validation.py`:
- `Scalar = str | int | float | bool` (module-level type alias).
- `class FormError(ValueError)` тАФ raised on any malformed value.
- `def validate_attribute_value(value: object) -> None` тАФ returns None if `value`
  is a `Scalar` or a `list` whose every element is a `Scalar`; otherwise raises
  `FormError` with a message naming the offending shape.
Acceptance:
- scalars pass: `"x"`, `""`, `0`, `3.14`, `True`;
- a flat list of scalars passes: `["a", "b"]`, `[1, 2]`, `[]` (empty list ok);
- nested/structured values raise FormError: a `dict`, a `list` containing a `list`,
  a `list` containing a `dict`, a `None`, a `tuple`, a set.
Note: `bool` is a `Scalar` deliberately (YAML true/false); do NOT special-case it
out even though `bool` is a subclass of `int`.
Constraints: pure function, no I/O, stdlib + the type alias only. Do not import
Pydantic here.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: the store validates the FORM of attributes only (well-typed:
scalar or list of scalars) and assigns no meaning to keys or values. (ADR-0001,
ADR-0002)
File scope: create `src/micro_entity/validation.py` and
`tests/test_validation.py` only. Do not touch anything else.

## 3. Entity model тАФ [todo]
Goal: the generic entity as a validated Pydantic model: an id, two timestamps, an
optional free-text prose body, and a bag of generic attributes. There is NO
structural `status` field тАФ status, when a profile uses it, is an ordinary key in
the attribute bag (see ADR-0004). The model validates FORM only.
Interface: in `src/micro_entity/entity.py`, a Pydantic v2 model `Entity` with:
- `id: str` тАФ must match `^[A-Za-z0-9_-][A-Za-z0-9._-]*$` (first character is not a
  dot; remaining characters are ASCII alphanumerics, dot, underscore, or hyphen)
  and be at most 200 characters; otherwise a validation error. This keeps every id
  usable directly as a filesystem-safe filename and as a portable key for any
  storage backend. Rejects empty, whitespace, path separators (`/`, `\`), `.`/`..`,
  leading-dot names, control characters, and non-ASCII.
- `created: datetime` and `updated: datetime` тАФ timezone-aware datetimes.
- `body: str | None = None` тАФ optional free-text prose (the document body region of
  a record). It is NOT an attribute and is NOT subject to attribute form-validation:
  any string (including empty string) or None is accepted. It is kept distinct from
  the attribute bag so that membership-query operates only on attributes and
  full-text search operates only on the body.
- `attributes: dict[str, Scalar | list[Scalar]]` тАФ every value validated via
  `validate_attribute_value` (item 2); a malformed value fails model construction.
  Attribute keys must be non-empty strings.
- The model is immutable (`model_config = ConfigDict(frozen=True)`).
Acceptance:
- a well-formed Entity constructs and exposes its fields;
- empty/whitespace id rejected; an id with a path separator (`/`), a leading dot,
  or non-ASCII rejected; an id over 200 characters rejected; a valid id like
  `ADR-0007` or `build-entity` accepted; empty attribute key rejected;
- `body` defaults to None when omitted; a string body (including "") is accepted and
  exposed verbatim; `body` is never treated as an attribute (it does not appear in
  `attributes` and is not run through `validate_attribute_value`);
- an attribute whose value is a nested dict/list rejects (delegates to item 2's
  validator тАФ wire it via a field validator, do not reimplement the check);
- a naive (tz-less) datetime for created/updated is rejected;
- the instance is frozen: attribute assignment after construction raises.
Constraints: reuse `validate_attribute_value` and `Scalar` from
`micro_entity.validation`; do not duplicate the value-shape logic. No I/O.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: an entity = an id + store-managed timestamps + an optional
prose body + a bag of generic attributes (key -> scalar | list); the server
validates form only and assigns no meaning to attribute keys or values. There is no
structural status field тАФ status is a designated attribute, not a core field.
(ADR-0001, ADR-0002, ADR-0004)
File scope: create `src/micro_entity/entity.py` and `tests/test_entity.py` only.
Do not modify `validation.py`; do not touch anything else.

## 4. Registered value-set validation тАФ [todo]
Goal: a pure, generic validator that checks an attribute's value(s) against a
caller-provided allowed set. This is the single mechanism behind both status enums
and any other cross-cutting attribute that must avoid typo-drift; the core knows no
specific attribute names or values. Status is merely its first consumer тАФ this
function has no notion of "status".
Interface: in `src/micro_entity/validation.py` (extend the existing module):
- `def validate_against_set(value: Scalar | list[Scalar], allowed: collections.abc.Set[Scalar]) -> None`
  тАФ returns None if every element of `value` is in `allowed` (a scalar is treated as
  a one-element set; a list must have all elements in `allowed`); otherwise raises
  `FormError` naming the offending value(s) and listing the allowed values.
Acceptance:
- a scalar present in the allowed set passes; a scalar absent raises FormError;
- a list whose every element is in the set passes; a list with any element absent
  raises FormError naming that element;
- an empty list value passes (no element violates the set тАФ vacuously true);
- an empty allowed set: a scalar or non-empty list always raises FormError; an empty
  list value still passes (nothing to check);
- membership is type-strict and exact (consistent with item 2's Scalar typing): e.g.
  `1` (int) is not matched by an allowed set containing only `"1"` (str);
- the error message includes the offending value(s) and the allowed values.
Constraints: pure function, no I/O. Do not bake in any concrete attribute names or
values (no "status", "done", "Accepted", etc.) тАФ both the value and the allowed set
are always passed in by the caller. Reuse the `Scalar` alias from item 2.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: a per-attribute registered value-set is generic form
validation applied to whatever attribute the caller chooses; the core assigns no
meaning to the attribute or its values. Status enums are one instance of this
mechanism, configured and enforced by the profile servers, never by the core.
(ADR-0001, ADR-0002, ADR-0004)
File scope: edit `src/micro_entity/validation.py` and `tests/test_validation.py`
only. Do not touch anything else.
