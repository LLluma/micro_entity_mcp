## 2. Attribute value form-validation — [todo]
Goal: a pure validator that enforces the generic attribute value shape the store
accepts — a scalar or a flat list of scalars, nothing nested. This is the "validate
form only, assign no meaning" rule for attribute VALUES.
Interface: in `src/micro_entity/validation.py`:
- `Scalar = str | int | float | bool` (module-level type alias).
- `class FormError(ValueError)` — raised on any malformed value.
- `def validate_attribute_value(value: object) -> None` — returns None if `value`
  is a `Scalar` or a `list` whose every element is a `Scalar`; otherwise raises
  `FormError` with a message naming the offending shape.
Acceptance:
- scalars pass: `"x"`, `""`, `0`, `3.14`, `True`;
- a flat list of scalars passes: `["a", "b"]`, `[1, 2]`, `[]` (empty list ok);
- nested/structured values raise FormError: a `dict`, a `list` containing a `list`,
  a `list` containing a `dict`, a `None`, a `tuple`, a set.
Note: `bool` is a `Scalar` deliberately (YAML true/false); do NOT special-case it
out even though `bool` is a subclass of `int`.
Constraints: pure function, no I/O, stdlib + the type alias only. Do not import
Pydantic here.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: the store validates the FORM of attributes only (well-typed:
scalar or list of scalars) and assigns no meaning to keys or values. (ADR-0001,
ADR-0002)
File scope: create `src/micro_entity/validation.py` and
`tests/test_validation.py` only. Do not touch anything else.

## 3. Entity model — [todo]
Goal: the generic entity as a validated Pydantic model: an id, a status, two
timestamps, and a bag of generic attributes. The model validates FORM only.
Interface: in `src/micro_entity/entity.py`, a Pydantic v2 model `Entity` with:
- `id: str` — must be non-empty and contain no whitespace; otherwise a validation
  error.
- `status: str` — any non-empty string here (the allowed SET is enforced elsewhere,
  not in this model); empty string is invalid.
- `created: datetime` and `updated: datetime` — timezone-aware datetimes.
- `attributes: dict[str, Scalar | list[Scalar]]` — every value validated via
  `validate_attribute_value` (item 2); a malformed value fails model construction.
  Attribute keys must be non-empty strings.
- The model is immutable (`model_config = ConfigDict(frozen=True)`).
Acceptance:
- a well-formed Entity constructs and exposes its fields;
- empty/whitespace id rejected; empty status rejected; empty attribute key rejected;
- an attribute whose value is a nested dict/list rejects (delegates to item 2's
  validator — wire it via a field validator, do not reimplement the check);
- a naive (tz-less) datetime for created/updated is rejected;
- the instance is frozen: attribute assignment after construction raises.
Constraints: reuse `validate_attribute_value` and `Scalar` from
`micro_entity.validation`; do not duplicate the value-shape logic. No I/O.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: an entity = an id + server-understood structural fields
(status, timestamps) + a bag of generic attributes (key -> scalar | list); the
server validates form only and assigns no meaning to attribute keys or values.
(ADR-0001, ADR-0002)
File scope: create `src/micro_entity/entity.py` and `tests/test_entity.py` only.
Do not modify `validation.py`; do not touch anything else.

## 4. Status set validation — [todo]
Goal: a pure validator that checks an entity status against a caller-provided
allowed set. The set is supplied per deployment profile; the core knows no specific
statuses. This is generic form validation, not domain knowledge.
Interface: in `src/micro_entity/validation.py` (extend the existing module):
- `def validate_status(status: str, allowed: collections.abc.Set[str]) -> None` —
  returns None if `status` is in `allowed`; otherwise raises `FormError` naming the
  bad status and listing the allowed values.
Acceptance:
- a status present in the allowed set passes;
- a status absent from the set raises FormError;
- an empty allowed set always raises FormError;
- the error message includes the offending status and the allowed values.
Constraints: pure function, no I/O. Do not bake in any concrete status names (no
"done", "Accepted", etc.) — the set is always passed in by the caller.
Done means: tests green AND `make lint` clean.
How to run: `make test` then `make lint`.
Governing decisions: status is a structural field validated against a per-profile
registered set; the core assigns no meaning to specific status values. (ADR-0001,
ADR-0002, ADR-0003 — the core stays dumb; concrete status enums live in the profile
servers, not the core)
File scope: edit `src/micro_entity/validation.py` and `tests/test_validation.py`
only. Do not touch anything else.
