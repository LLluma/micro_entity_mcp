# TODO-0008 — per-project directory partitioning for both entity stores
ADR: ADR-0006
Status: active — 7 items (3 done, 4 todo)

Implements ADR-0006: both `todo` and `adr` stores partition per-project by
directory (`base/<segment>`), segment resolved as explicit `project` arg →
workspace-derived default (`basename(cwd)`) → hard-fail. Partitioning lives in the
neutral core (`micro_entity`); servers translate a `project` arg into a segment.
Items ordered by dependency: A (pure helpers) → store → provider → server wiring →
entrypoints. Baseline at plan time: 261 tests green, ruff + pyright clean.

Config (orchestrator, not a code item): opencode.jsonc `--directory`→`--project`
for both servers so the process cwd is the workspace. Ops (user): restart servers;
move existing flat data under a `<segment>` subdir.

## 1. sanitize_segment — slugify a raw name to one safe path component — [done]
Goal: reduce an arbitrary project/segment name to a single, filesystem-safe path
component that can never traverse outside a base directory.
Interface: `def sanitize_segment(name: str) -> str` in new module
`src/micro_entity/partition.py`. Sync, pure, stdlib only.
Rule: lowercase; keep only `[a-z0-9._-]`; replace every maximal run of other
characters (including `/` and whitespace) with a single `-`; strip leading/trailing
characters in `-._`; and ensure the result is never `.` or `..` and contains no
`..` substring (neutralize by replacement). Return `""` when nothing survives.
Acceptance:
- `"MyProject"` → `"myproject"`; `"micro_entity_mcp"` → `"micro_entity_mcp"`.
- `"a/b\\c"` and `"a   b"` → single `-` joins (`"a-b-c"`, `"a-b"`).
- `"../etc"` and `".."` and `"."` → a value that is not `.`/`..` and has no `..`
  (e.g. `"etc"` / `""`); leading/trailing `.-_` stripped (`"__x__"` → `"x"`).
- `""` → `""`; a string of only separators/punctuation → `""`.
- `"Feature #1!"` → `"feature-1"`.
Constraints: pure function, no I/O, no dependence on `os`/filesystem. Stdlib only
(`re` is fine).
Done means: tests green AND ruff/pyright clean.
How to run: `make test`, `make lint` (see Makefile).
File scope: create `src/micro_entity/partition.py` and
`tests/test_partition.py` only; do not delete/move/rename any file; touch nothing
else.

## 2. resolve_segment — apply precedence explicit → workspace → None — [done]
Goal: choose the effective directory segment from an optional explicit override and
an optional workspace path, applying slugification and precedence.
Interface: `def resolve_segment(*, explicit: str | None, workspace: str | None) ->
str | None` in `src/micro_entity/partition.py`. Sync, pure.
Behavior: return `sanitize_segment(explicit)` if `explicit` is given and slugifies
to a non-empty string; else return `sanitize_segment(basename(workspace))` if
`workspace` is given and its basename slugifies non-empty; else return `None`.
Use `os.path.basename` (or `pathlib`) to take the last path component of
`workspace` before slugifying; a trailing slash must not yield an empty basename
(strip it first). `None` and `""` inputs are both "absent".
Acceptance:
- explicit `"Shared"`, workspace `"/home/u/proj"` → `"shared"` (explicit wins).
- explicit `None`, workspace `"/home/u/Micro_Entity_MCP"` → `"micro_entity_mcp"`.
- explicit `None`, workspace `"/home/u/proj/"` (trailing slash) → `"proj"`.
- explicit `"  "` (slugifies empty), workspace `"/x/proj"` → `"proj"` (falls
  through past an empty explicit).
- explicit `None`, workspace `None` → `None`; explicit `None`, workspace `"/"` →
  `None` (basename empty).
Constraints: pure; may use `os.path`/`pathlib` for the basename only — no cwd
lookup, no filesystem access. Reuse `sanitize_segment`; do not reimplement slugging.
Done means: tests green AND ruff/pyright clean.
How to run: `make test`, `make lint`.
File scope: `src/micro_entity/partition.py`, `tests/test_partition.py` only; do not
delete/move/rename anything; touch nothing else.

## 3. MarkdownStore optional segment — resolve dir to base/segment — [done]
Goal: let a MarkdownStore be bound to `base/<segment>` without changing any existing
caller that passes only a directory.
Interface: change `MarkdownStore.__init__` in
`src/micro_entity/markdown_store.py` to
`def __init__(self, directory, *, segment: str | None = None, clock=...)`. When
`segment` is a non-empty string, the store's resolved directory becomes
`Path(directory).resolve() / segment`; when `segment` is `None` or `""`, behavior
is exactly as today (`Path(directory).resolve()`). The existing `mkdir(parents=True,
exist_ok=True)` then creates the resolved (possibly nested) directory.
Acceptance:
- `MarkdownStore(tmp)` (no segment) writes/reads directly under `tmp` — every
  existing test still passes unchanged.
- `MarkdownStore(tmp, segment="proj")` resolves its directory to `tmp/proj`,
  creates it, and a `create(...)` writes the file under `tmp/proj/`.
- `MarkdownStore(tmp, segment="")` behaves identically to no segment (flat `tmp`).
- `_path_for` still confines paths within the resolved (segmented) directory.
Constraints: `segment` is keyword-only with default `None` so no positional caller
breaks. Do NOT add project/domain vocabulary to the core — this is a generic
directory segment. Do not change `_path_for`/write/read logic beyond the base
directory it resolves against. Do not import `partition.py` here (this item is only
the segment parameter; segment VALUES are computed by callers).
Done means: full suite green (existing 261 unchanged + new) AND ruff/pyright clean.
How to run: `make test`, `make lint`.
File scope: `src/micro_entity/markdown_store.py` and
`tests/test_markdown_store.py` (or a new file under `tests/markdown_store/`) only;
do not delete/move/rename any existing test file; touch nothing else.

## 4. StoreProvider — per-segment store cache with hard-fail — [todo]
Goal: resolve a per-operation `project` (or the workspace default) to a
MarkdownStore bound to that segment, caching one store per resolved segment, and
refuse to operate when no segment can be resolved.
Interface: `class StoreProvider` in `src/micro_entity/partition.py`:
- `def __init__(self, base: Path, default_segment: str | None) -> None`
- `def get(self, project: str | None = None) -> MarkdownStore`
`get` computes the effective segment as `resolve_segment(explicit=project,
workspace=None)` if that is non-empty, else `self._default_segment`; if the result
is `None` or empty it raises `UnresolvedSegmentError` (a new exception defined in
this module). Otherwise it returns `MarkdownStore(self._base, segment=<seg>)`,
constructing it once per distinct segment and caching it (repeat calls with the same
resolved segment return the same instance).
Acceptance:
- `StoreProvider(base, "def").get()` → store bound to `base/def`; called twice →
  same instance.
- `.get("Other")` → store bound to `base/other`; distinct instance from the default.
- `.get("  ")` (explicit slugifies empty) → falls back to `default_segment`.
- `StoreProvider(base, None).get()` (no default, no override) raises
  `UnresolvedSegmentError`; `StoreProvider(base, None).get("proj")` → `base/proj`.
Constraints: reuse `resolve_segment`/`sanitize_segment`; do not duplicate slugging.
The provider does the hard-fail (ADR-0006); `MarkdownStore` stays permissive.
`default_segment` is assumed already-resolved (the entrypoint computes it via
`resolve_segment`); still guard against it being empty. Pure aside from constructing
stores; no cwd lookup inside the provider.
Done means: full suite green AND ruff/pyright clean.
How to run: `make test`, `make lint`.
File scope: `src/micro_entity/partition.py`, `tests/test_partition.py` only; do not
delete/move/rename anything; touch nothing else.

## 5. todo server — route tools through a StoreProvider + `project` arg — [todo]
Goal: make the todo server resolve its store per call from a StoreProvider, exposing
an optional `project` override on every data tool.
Interface: change `build_server` in `src/servers/todo.py` to
`def build_server(provider: StoreProvider) -> FastMCP`. Every data tool
(`create`, `get`, `list`, `query`, `update`, `delete`, `next`, `clear`,
`is_complete`) gains a trailing parameter `project: str | None = None` and, as its
first action, resolves `store = provider.get(project)`, mapping a raised
`UnresolvedSegmentError` to `ToolError`. `health` is unchanged (no `project`). All
helper calls (`_next_order`, `_next_id`, etc.) use the per-call `store`.
Acceptance:
- With a provider over `tmp` default segment `"proj"`, `create(...)` writes under
  `tmp/proj/`; `list()`/`get()`/`next()` read from there.
- Passing `project="other"` to any tool targets `tmp/other/` and is isolated from
  the default segment's records.
- A provider that resolves no segment (no default, no `project`) → the tool raises
  `ToolError`.
- Existing todo-server test behaviors still hold once their `_client` helper builds
  a provider (see File scope); tool semantics (reserved-key screen, status
  validation, ordering, `next`, `is_complete`) unchanged.
Constraints: keep tool names and existing semantics identical; only add the
`project` parameter and per-call store resolution. Update the module-level entrypoint
minimally so the module still imports (full entrypoint wiring is item 7 — a
temporary `StoreProvider(Path(TODO_DIR), None)` or similar is acceptable here and
finalized in item 7). Do NOT touch `adr.py`.
Update `tests/todo_server/conftest.py` `_client(tmp_path)` helper to build the
server via a `StoreProvider` with a fixed default segment so existing tests pass
without per-test edits; add tests for the new `project` override and the
hard-fail→ToolError path.
Done means: full suite green AND ruff/pyright clean.
How to run: `make test`, `make lint`.
File scope: `src/servers/todo.py`, `tests/todo_server/conftest.py`, and files under
`tests/todo_server/` only; do not delete/move/rename existing test files except as
needed to update the shared conftest; touch nothing in `adr.py` or `micro_entity`.
