# TODO-0003 — micro_entity: Store protocol + MarkdownStore
ADR: ADR-0001, ADR-0003
Status: active — 8 items (7 done, 1 todo)

Settled decisions for this batch (from session clarification):
- One concrete `MarkdownStore` serves BOTH profiles; partition = a directory, one
  file per entity named `{id}.md`. `id` = filename stem (round-trips to the Entity).
- Store-managed timestamps via an INJECTABLE CLOCK: the store takes
  `clock: Callable[[], datetime] = lambda: datetime.now(UTC)`. `create` sets
  `created` + `updated`; `update` bumps `updated` only.
- QUARANTINE load model: `list()` returns `(valid_entities, errors)`, NOT a bare
  list. A record that fails parse/validation on load is skipped and reported in
  `errors` (id + reason), never fatal.
- Atomic writes: temp-file + `os.replace` (no corruption on a torn write). No locking
  (single-user, per-DIR).
- LOSSLESS update: load the existing frontmatter `CommentedMap`, patch only the
  changed keys, bump `updated`, re-serialize — comments/order preserved. Do NOT
  rebuild from the Entity on update.
- Store is CAPABLE of delete/clear. Append-only is NOT enforced here — it is the adr
  profile's tool-surface discipline (TODO-0006).
- Registered value-set enforcement on load/write is driven by config PASSED IN by the
  profile; the store/core bakes in no concrete attribute names or values.

## 2. MarkdownStore skeleton: partition + path resolution — [done]

- `def _path_for(self, id: str) -> Path` — returns `directory / f"{id}.md"`; validates

## 3. Atomic write helper — [done]
Goal: write file content so a crash mid-write never leaves a corrupt record.
Interface: `def _atomic_write(self, path: Path, text: str) -> None` — write to a
temp file in the same directory, flush, then `os.replace` onto `path`.
Acceptance: writing creates the file with exact content; overwriting replaces atomically
(no temp file left behind); content with unicode + trailing newline preserved.
Constraints: temp file in the same dir (so replace is atomic on one filesystem).
Done means: tests green AND `make lint` clean.
File scope: edit `markdown_store.py` and its test only.

## 4. create — [done]
Goal: persist a new record, stamping both timestamps from the clock.
Interface: `def create(self, id: str, *, attributes: dict[str, Scalar | list[Scalar]], body: str | None = None) -> Entity`
— builds an Entity with `created = updated = clock()`, renders via `entity_to_parts`
+ `serialize_document`, atomically writes `{id}.md`. Raises `FileExistsError` (or a
store-specific `AlreadyExists`) if the file already exists.
Acceptance: create writes a parseable file; the returned Entity has equal created/updated
from the injected clock; creating an existing id raises; attributes round-trip; a bad
attribute value raises before any file is written.
Constraints: reuse codec (TODO-0002) + clock; no status logic (status is just an
attribute supplied by the caller).
Done means: tests green AND `make lint` clean.
File scope: edit `markdown_store.py` and its test only.

## 6. list (quarantine) — [done]
(Implemented as `load_all()` — name chosen to avoid shadowing builtin `list`; matches the Store protocol seam.)
Goal: load every record in the partition, skipping and reporting malformed ones.
Interface: `def list(self) -> tuple[list[Entity], list[LoadError]]` — iterate
`*.md` in the directory; each that parses/validates → entities; each that fails →
a `LoadError(id=<stem>, reason=<message>)`.
Acceptance: a directory of N valid files → N entities, no errors; a mix → valid ones
returned plus a LoadError per bad file (with id + reason); an empty/absent directory →
([], []); ordering of returned entities is deterministic (sorted by id).
Constraints: never raise on a single bad record; reuse get's parse path.
Done means: tests green AND `make lint` clean.
File scope: edit `markdown_store.py` and its test only.

## 7. update (lossless patch) — [done]
Goal: change attributes/body of an existing record while preserving hand-edits
(comments, key order) and bumping `updated`.
Interface: `def update(self, id: str, *, attributes: dict[str, Scalar | list[Scalar]] | None = None, body: str | None | UnsetType = UNSET) -> Entity`
— load the existing frontmatter CommentedMap, patch only the provided attribute keys
(add/replace), set `updated = clock()`, optionally replace the body, atomically write.
Returns the updated Entity. Raises NotFound if absent.
Acceptance: updating one attribute preserves all other keys, their order, and any
comments in the file (assert against a fixture with comments); `updated` advances while
`created` is unchanged; body left untouched when `body` is UNSET, replaced when given;
a bad new attribute value raises before writing; updating a missing id raises.
Constraints: MUST patch the loaded CommentedMap (do NOT rebuild via entity_to_parts);
reuse codec + clock. Provide an `UNSET` sentinel so "don't touch body" differs from
"set body to None".
Done means: tests green AND `make lint` clean.
File scope: edit `markdown_store.py` and its test only.

## 8. delete + clear — [todo]
Goal: remove one record, or empty the whole partition (ephemeral discard).
Interface: `def delete(self, id: str) -> None` (raise NotFound if absent);
`def clear(self) -> None` (remove all `*.md` in the partition).
Acceptance: delete removes the file; deleting a missing id raises NotFound; clear
removes every record file but leaves the directory; clear on an empty partition is a
no-op.
Constraints: only touch `*.md` records in the partition; never recurse outside it.
Done means: tests green AND `make lint` clean.
File scope: edit `markdown_store.py` and its test only.
