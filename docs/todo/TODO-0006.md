# TODO-0006 — adr profile server (durable)
ADR: ADR-0002, ADR-0004, ADR-0005
Status: active — items TBD (specs written when batch is reached)

Settled decisions for this batch (items H-adr/Fork-3, I, J, K):
- Thin FastMCP stdio server `servers/adr.py` over the shared core + MarkdownStore.
  Its own `ADR_DIR` (env-configured), centralized, cross-cutting, out of client
  repos. Dogfoods `docs/adr/` (the files already match the schema). Core stays dumb.
- SCHEMA (ADR-0005): rich-frontmatter on-disk shape is canonical. Per record:
  id = filename stem; created/updated = structural timestamps; body = prose (search
  target); attribute bag = title, tags, supersedes, superseded_by, refines,
  refined_by, relates_to, and any future cross-cutting key.
- STATUS enum (ADR-0004, ADR-0005): `{Proposed, Accepted, Superseded}`, registered
  required value-set. Core knows none of these values.
- APPEND-ONLY (ADR-0002, item C): server exposes NO delete tool. A changed decision
  = a new record + old record set to `Superseded`. Status transitions + hand-edits
  allowed (lossless).
- SUPERSESSION (ADR-0005): an attribute pointer, not a status string. Superseded
  record: `status: Superseded` + `superseded_by: ADR-NNNN`; superseding record:
  `supersedes: ADR-NNNN`. Never the `Superseded by ADR-NNNN` baked form.
- TIMESTAMP MIGRATION (ADR-0005): on load, if created/updated absent, derive both
  from `date` (midnight UTC); store writes explicit created/updated thereafter.
  Schema-violating records quarantined + reported, never fatal.
- SEARCH (item I): full-text v1 = case-INSENSITIVE substring over body + frontmatter
  attribute values; returns full records (wide dumb window, ADR-0002). Orthogonal to
  `query` (exact attribute membership). May graduate to vector (mc_rag_doc) later —
  deferred, external.
- Tools (one atomic item each): server skeleton; add; get; list (quarantine errors);
  update(status); supersede helper; query (TODO-0004); search.
- Errors (item J): structured MCP tool errors; quarantine surfaced. Tests (item K):
  FastMCP in-memory client.
- Reconcile on-disk ADR-0001..0005 to the final schema (add created/updated, confirm
  status values) as part of this batch, OR rely on load-time migration — decide when
  reached.

NOTE: the orchestration `adr-format.md` convention is being retired and is NOT a
consumer of this store (ADR-0005).
