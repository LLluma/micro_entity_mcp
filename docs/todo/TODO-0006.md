# TODO-0006 — adr profile server (durable)
ADR: ADR-0002, ADR-0004, ADR-0005
Status: active — 10 items (6 done, 4 todo)

Settled decisions for this batch (items H-adr/Fork-3, I, J, K; see ADR-0005):
- Thin FastMCP stdio server `src/servers/adr.py` over the shared core + MarkdownStore,
  its own `ADR_DIR` (env-configured), centralized/out-of-client-repos. Dogfoods
  `docs/adr/`. Core stays dumb.
- SCHEMA (ADR-0005): rich-frontmatter on-disk shape is canonical. id = filename stem;
  created/updated = structural timestamps; body = prose (search target); attribute bag
  = title, tags, supersedes, superseded_by, refines, refined_by, relates_to, and any
  future cross-cutting key.
- STATUS enum: `{Proposed, Accepted, Superseded}`, required value-set.
- APPEND-ONLY: server exposes NO delete tool. A changed decision = a new record + the
  old set to `Superseded`.
- SUPERSESSION: attribute pointer, not a status string. Superseded record:
  `status: Superseded` + `superseded_by: ADR-NNNN`; superseding record:
  `supersedes: ADR-NNNN`. Never the `Superseded by ADR-NNNN` baked form.
- TIMESTAMP MIGRATION: on load, if created/updated absent, derive both from `date`
  (midnight UTC); store writes explicit created/updated thereafter.
- SEARCH (I): full-text v1 = case-INSENSITIVE substring over body + frontmatter
  attribute values; returns full records. Orthogonal to `query`. Vector deferred.
- Errors (J): structured MCP tool errors; quarantine surfaced. Tests (K): in-memory
  client.

## 1. Server skeleton + config + test harness — [done]
Goal: a runnable FastMCP adr server bound to a MarkdownStore over `ADR_DIR`, with the
status value-set and attribute-key constants, plus an in-memory client fixture.
Interface: `src/servers/adr.py` exposing FastMCP app `mcp`; `ADR_DIR` from env;
constants `STATUS_VALUES = {"Proposed","Accepted","Superseded"}`, `STATUS_KEY`,
`SUPERSEDES_KEY="supersedes"`, `SUPERSEDED_BY_KEY="superseded_by"`. Pytest fixture with
a `tmp_path` store via in-memory client.
Acceptance: app starts; a list stub returns; constants present.
File scope: create `src/servers/adr.py` and `tests/test_adr_server.py` only.

## 2. Load normalizer: timestamp migration — [done]
Goal: make legacy records (with `date:` but no created/updated) loadable by deriving
the structural timestamps before Entity construction.
Interface: a load path / helper `def _normalize_frontmatter(fm) -> fm` applied during
the server's list/get: if `created`/`updated` missing but `date` present, set both to
`date` at midnight UTC; leave already-timestamped records untouched.
Acceptance: a record with only `date` loads with created==updated==midnight(date) UTC;
a record with explicit created/updated is unchanged; a record with neither date nor
timestamps quarantines with a clear reason.
Constraints: migration is adr-PROFILE policy, not core; do not modify the core codec.
File scope: edit `src/servers/adr.py` and its test only.

## 7. supersede tool — [todo]
