# TODO-0004 — micro_entity: membership query
ADR: ADR-0001
Status: complete — 2 items (2 done, 0 todo)

Settled decisions for this batch (item D, fully ratified):
`query({attr: [values], ...})` is a primitive membership filter returning full
entities. Within an attribute: OR (set intersection). Across attributes: AND. No
expression tree; NOT / cross-attr OR / diff are composed by the caller.

Edge-case semantics (all ratified):
1. Entity LACKS the queried attribute → excluded (empty intersection).
2. Scalar attribute value → treated as a 1-element set.
3. List attribute value vs query list → match iff intersection != empty (within-attr OR).
4. `query({})` (no constraints) → returns ALL (vacuous AND = true); doubles as "list all".
6. Type-strict, exact equality: `1`(int) != `"1"`(str), `1` != `1.0`, `True` != `1`.
9. Strings: case-SENSITIVE exact match (fuzzy/case-insensitive is `search`'s job).
D-i. Empty value list `query({tags: []})` → matches NOTHING (OR over zero values =
   false → that clause fails → entity excluded).
D-ii. RESOLVED by ADR-0004: `status` is an attribute, so `query({status:[...]})`
   works with no special case. `query` operates over the attribute bag ONLY; the
   structural fields (id, timestamps) and `body` are NOT membership-queryable.

## 1. Single-attribute membership match — [done]
Goal: decide whether one entity matches one attribute criterion, with the ratified
scalar/list, type-strict, missing-key, empty-values semantics.
Interface: in `src/micro_entity/query.py`:
- `def match_attribute(entity: Entity, key: str, values: list[Scalar]) -> bool` —
  True iff the entity has attribute `key` and its value(s), treated as a set,
  intersect `values` (type-strict). A scalar attribute value is a 1-element set.
Acceptance:
- scalar attribute equal to a query value → True; unequal → False;
- list attribute sharing >=1 value with the query list → True; disjoint → False;
- missing attribute key → False;
- empty `values` (`[]`) → False (OR over zero values);
- type-strict: attribute `1` (int) vs `values=["1"]` (str) → False; `1` vs `[1.0]` →
  False; `True` vs `[1]` → False;
- case-sensitive: `"Tag"` vs `["tag"]` → False.
Constraints: pure; reuse `Entity`, `Scalar`. No status/body special-casing — operate
on `entity.attributes` only.
Done means: tests green AND `make lint` clean.
File scope: create `src/micro_entity/query.py` and `tests/test_query.py` only.

## 2. Conjunctive multi-attribute query — [done]
Goal: filter a collection of entities by AND-across-attributes, returning full
entities; the empty criteria case returns all.
Interface: in `src/micro_entity/query.py`:
- `def query(entities: Iterable[Entity], criteria: Mapping[str, list[Scalar]]) -> list[Entity]`
  — returns entities for which `match_attribute` is True for EVERY key in `criteria`
  (AND). `criteria == {}` returns all entities.
Acceptance:
- two criteria → only entities matching both (AND); an entity matching one but not the
  other is excluded;
- within a single criterion the values OR (matches any) — via item 1;
- `query(es, {})` → all entities; `query([], {...})` → [];
- one criterion with an empty value list → no entity matches (excludes all), even if
  other criteria would match;
- result order is the input order (stable), deduped not required (inputs unique by id).
Constraints: pure; build on `match_attribute` (item 1); do not reimplement membership.
Done means: tests green AND `make lint` clean.
File scope: edit `src/micro_entity/query.py` and `tests/test_query.py` only.
